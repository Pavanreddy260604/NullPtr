Question 1. c) i) Discuss about the data types in Python with examples. ii) What are benefits of python? How to setup python?

Data Types in Python

Python has several built-in data types used to classify the type of data a variable holds. Since Python is dynamically typed, you do not need to declare the type explicitly.

Python is dynamically typed, which means you don't need to declare variable types explicitly. The type is inferred at runtime.
Numeric Types:
  • Integer (int): Whole numbers without a fractional part.
    Example: x = 10
  • Float (float): Numbers with decimal points.
    Example: y = 10.5
  • Complex (complex): Numbers with a real and imaginary part (a+bj).
    Example: z = 2 + 3j
Sequence Types:
  • String (str): Ordered sequence of characters enclosed in quotes.
    Example: name = "IoT"
  • List (list): Ordered, mutable collection of items.
    Example: fruits = ["apple", "banana", "cherry"]
  • Tuple (tuple): Ordered, immutable collection of items.
    Example: coordinates = (10.0, 20.0)
Mapping Type:
  • Dictionary (dict): Unordered collection of key-value pairs.
    Example: student = {"name": "John", "age": 20}
Boolean Type:
  • Boolean (bool): Represents True or False.
    Example: is_active = True
Set Type:
  • Set (set): Unordered collection of unique items.
    Example: unique_ids = {1, 2, 3}

Benefits of Python

  • Simple and Easy to Learn: Python has a clean syntax similar to English.
  • Interpreted Language: Code is executed line by line, making debugging easier.
  • Portable/Platform Independent: Python code can run on Windows, macOS, and Linux without modification.
  • Rich Standard Library: Extensive libraries for web development, data science (NumPy, Pandas), and IoT.
Python's simplicity and extensive libraries make it one of the most popular programming languages for beginners and experts alike.

How to Setup Python

  1. Download: Visit the official website python.org and download the latest version for your OS.
  2. Install: Run the installer. Crucial Step: Check the box that says "Add Python to PATH" during installation to run Python from the command line.
  3. Verify: Open a terminal (Command Prompt) and type python --version to verify the installation.

Question 2. c) i) Describe different access modes of the files with an example. ii) List different operators in python, in the order of precedence.

File Access Modes

When opening a file using open(), you must specify the mode.

Mode Description
'r' Read Only. Default mode. Pointer at the beginning. Error if file doesn't exist.
'w' Write Only. Overwrites file if it exists. Creates a new file if it doesn't.
'a' Append. Writes data to the end of the file. Creates a new file if it doesn't exist.
'r+' Read and Write. Pointer at the beginning.
'w+' Write and Read. Overwrites existing file.
'b' Binary Mode. Used for non-text files (images, audio) e.g., 'wb', 'rb'.
Be careful with 'w' mode as it will overwrite the entire file if it already exists.
Example:
# Opening a file in write mode file = open("example.txt", "w") file.write("Hello IoT") file.close()

Operators in Order of Precedence (Highest to Lowest)

Operator precedence determines which operation is performed first in an expression.

  1. Parentheses: ()
  2. Exponentiation: **
  3. Unary plus/minus, Bitwise NOT: +x, -x, ~x
  4. Multiplication, Division, Floor Division, Modulus: *, /, //, %
  5. Addition, Subtraction: +, -
  6. Bitwise Shifts: <<, >>
  7. Bitwise AND: &
  8. Bitwise XOR: ^
  9. Bitwise OR: |
  10. Comparison Operators: ==, !=, >, >=, <, <=
  11. Logical NOT: not
  12. Logical AND: and
  13. Logical OR: or
When in doubt about operator precedence, use parentheses to make your code more readable and ensure the correct order of operations.

Question 4. c) Explain How exception handling is done in python?

Exception Handling in Python

Exception handling allows a program to respond to runtime errors (like dividing by zero or file not found) without crashing. Python uses the try, except, else, and finally blocks.

Proper exception handling makes your programs more robust and user-friendly by gracefully handling errors instead of crashing.
Structure:
  • try block: Contains the code that might raise an exception
  • except block: Contains the code to handle the error if one occurs in the try block
  • else block (optional): Executed only if no exceptions were raised
  • finally block (optional): Executed always, regardless of whether an exception occurred (useful for cleanup like closing files)
Example:
try: numerator = 10 denominator = 0 result = numerator / denominator except ZeroDivisionError: print("Error: You cannot divide by zero.") else: print("Division successful:", result) finally: print("Execution completed.")

Question 4. d) Illustrate the different types of control flow statements in python with flow charts?

Control Flow Statements

Control flow statements determine the order in which code is executed.

1. Conditional Statements (Decision Making)

Used to execute code based on conditions.

  • if statement: Executes if condition is true
  • if...else statement: Executes one block if true, another if false
  • elif: Used to check multiple conditions
Concept: Diamond shape represents the decision. If True → Path A. If False → Path B.
2. Iterative Statements (Loops)

Used to repeat a block of code.

  • for loop: Iterates over a sequence (list, string, range)
  • while loop: Repeats as long as a condition is true
Concept: Check Condition → If True, Execute Body and Loop back → If False, Exit.
3. Transfer Statements
  • break: Exits the loop immediately
  • continue: Skips the current iteration and moves to the next
  • pass: A null operation (placeholder)

Question 5. d) Explain modules and packages in python? Write advantages in python language?

Modules

A module is simply a Python file (with a .py extension) containing definitions and statements (functions, variables, classes).

Usage: You use the import keyword to use a module.

Example: import math allows you to use math.sqrt().

Packages

A package is a directory (folder) that contains multiple Python modules.

It must contain a special file called __init__.py (can be empty) to be recognized as a Python package.

Packages allow for a hierarchical structuring of the module namespace.

Structure Example:
MyPackage/ __init__.py module1.py module2.py

Advantages in Python Language

  • Code Reusability: Write code once in a module and import it into multiple programs
  • Organization: Helps organize large codebases into manageable, logical sections (files and folders)
  • Namespace Management: Avoids variable name clashes (e.g., moduleA.func() vs moduleB.func())
  • Maintainability: Easier to debug and update specific parts of the system without affecting the whole program

Question 6. c) Explain following: 1. HTTP Lib 2. URL Lib 3. SMTP Lib

These are standard Python libraries used for networking and internet protocols.

1. HTTP Lib (http.client in Python 3)

  • This module defines classes which implement the client side of the HTTP and HTTPS protocols
  • It is rarely used directly; most developers use higher-level libraries like requests or urllib
  • Function: It is used to open connections to web servers and send GET/POST requests manually

2. URL Lib (urllib)

  • urllib is a package that collects several modules for working with URLs. It is more high-level than http.client
  • Key components:
    • urllib.request: For opening and reading URLs (fetching data)
    • urllib.parse: For parsing URLs into components (scheme, netloc, path)
    • urllib.error: For containing the exceptions raised by urllib.request
  • Use Case: Downloading a file from the web or scraping a webpage

3. SMTP Lib (smtplib)

  • SMTP stands for Simple Mail Transfer Protocol
  • The smtplib module defines an SMTP client session object that can be used to send mail to any Internet machine with an SMTP or ESMTP listener daemon
  • Use Case: Sending emails (including attachments) programmatically from a Python script (e.g., sending an alert when an IoT sensor detects a fire)
These libraries provide the foundation for many network operations in Python, though developers often use higher-level libraries built on top of them for simpler interfaces.

Question 1. c) Explain the IoT Physical Devices and Endpoints.

IoT Physical Devices and Endpoints

IoT physical devices and endpoints differ from standard computers like desktops or laptops. They are generally specialized devices bridging the gap between the physical world and the digital world.

The "Thing" in IoT:

The fundamental block is the device itself, often called a "mote" or "node." It usually consists of:

  • Sensors: To collect data (temperature, motion, light)
  • Actuators: To perform actions (turn on a motor, switch a light)
  • Processing Unit: A microcontroller (like Arduino) or a Single Board Computer (SBC like Raspberry Pi) that processes data locally
  • Communication Interface: Modules to transmit data (Wi-Fi, Bluetooth, Zigbee, LoRa)
  • Power Source: Batteries, solar cells, or mains power, depending on the deployment
Endpoint Function:

The "endpoint" is the specific interface where the device connects to the network, allowing it to be addressable (usually via an IP address in IP-based networks) to send telemetry or receive commands.

IoT endpoints can range from simple sensors with minimal processing power to sophisticated devices like Raspberry Pi that can run complex applications locally.

Question 1. d) Enumerate on Raspberry Pi interfaces.

Raspberry Pi Interfaces

The Raspberry Pi is a Single Board Computer (SBC) equipped with various hardware interfaces to connect peripherals and sensors:

  • GPIO (General Purpose Input/Output): The 40-pin header used to connect sensors, LEDs, and switches. These pins can be programmed as input or output.
  • USB Ports (2.0/3.0): For connecting keyboards, mice, Wi-Fi dongles, or external drives.
  • HDMI (High-Definition Multimedia Interface): For video and audio output to a monitor or TV.
  • Ethernet Port (RJ45): For wired network/internet connection.
  • CSI (Camera Serial Interface): A ribbon cable port specifically for connecting the Raspberry Pi Camera Module.
  • DSI (Display Serial Interface): A ribbon cable port for connecting touch screen displays.
  • Audio Jack: 3.5mm jack for audio output (and composite video on older models).
  • Power Input: USB-C (on Pi 4) or Micro-USB (on older models) for power supply.
Always check the pinout diagram for your specific Raspberry Pi model before connecting peripherals to avoid damage to the device.

Strengths and Limitations of Raspberry Pi

Strengths:
  • High Processing Power: Compared to microcontrollers (like Arduino), the Pi has a full CPU (ARM-based) and RAM, capable of running complex algorithms and local data processing.
  • Full Operating System: It runs Linux (Raspberry Pi OS), allowing multitasking, file management, and the use of high-level languages like Python easily.
  • Connectivity: Built-in Wi-Fi, Bluetooth, and Ethernet make it ready for IoT networking out of the box.
  • Community Support: Massive global community providing libraries, tutorials, and troubleshooting.
Limitations:
  • Power Consumption: It consumes more power than a microcontroller, making it less suitable for battery-operated devices that need to run for months.
  • No Real-time Capability: As it runs a general-purpose OS (Linux), it is not strictly "real-time" (deterministic), which can be an issue for precise timing applications without additional hardware.
  • Cost: More expensive than simple microcontrollers like the ESP8266 or Arduino Nano.

IoT Physical Servers and Cloud Offerings

This refers to the backend of an IoT system where data is stored and analyzed.

Physical Servers:

Dedicated hardware set up locally or in a data center to handle IoT data. This requires maintenance, cooling, and security management by the user.

Cloud Offerings (PaaS/SaaS):

Major providers offer managed IoT platforms:

  • AWS IoT Core: Connects devices to the cloud, manages device shadows, and routes messages.

Cloud offerings significantly reduce the infrastructure management burden for IoT projects, allowing developers to focus on application logic rather than server maintenance.

Question 2. d) Write python program for sending Email when control is switched on in Raspberry.

Email Notification on Switch Press

Logic: Connect a switch to a GPIO pin. When the switch is pressed (Input goes HIGH/LOW), use Python's smtplib to send an email.

import RPi.GPIO as GPIO import smtplib import time # --- Configuration --- SWITCH_PIN = 18 SENDER_EMAIL = "your_email@gmail.com" SENDER_PASSWORD = "your_app_password" # Use App Password if using Gmail RECEIVER_EMAIL = "target_email@gmail.com" # --- GPIO Setup --- GPIO.setmode(GPIO.BCM) GPIO.setup(SWITCH_PIN, GPIO.IN, pull_up_down=GPIO.PUD_UP) # Button connects to Ground def send_email(): try: server = smtplib.SMTP('smtp.gmail.com', 587) server.starttls() server.login(SENDER_EMAIL, SENDER_PASSWORD) message = "Subject: Alert\n\nControl Switch turned ON!" server.sendmail(SENDER_EMAIL, RECEIVER_EMAIL, message) server.quit() print("Email Sent Successfully!") except Exception as e: print(f"Failed to send email: {e}") print("Waiting for switch press...") try: while True: input_state = GPIO.input(SWITCH_PIN) if input_state == False: # Button Pressed (Active Low) print("Switch Detected!") send_email() time.sleep(2) # Debounce delay except KeyboardInterrupt: GPIO.cleanup()
For Gmail, you'll need to generate an App Password instead of using your regular password due to security settings. Enable 2-factor authentication first, then generate an App Password in your Google Account settings.

Popular Raspberry Pi Commands

Since Raspberry Pi runs on Linux, it uses standard Linux terminal commands:

  • ls: Lists the files and folders in the current directory.

Question 5. c) i) Write a Python code for Interfacing LED and switch with Raspberry Pi. ii) Explain the SPI Raspberry Pi interface in detail.

Python Code for LED and Switch

Objective: Turn the LED ON when the button is pressed.

import RPi.GPIO as GPIO import time # Pin Definitions LED_PIN = 23 BUTTON_PIN = 24 # Setup GPIO.setmode(GPIO.BCM) GPIO.setup(LED_PIN, GPIO.OUT) GPIO.setup(BUTTON_PIN, GPIO.IN, pull_up_down=GPIO.PUD_UP) # Pull-up: Input is High by default print("Press the button to turn on LED") try: while True: button_state = GPIO.input(BUTTON_PIN) if button_state == False: # Button Pressed (Ground connection) GPIO.output(LED_PIN, True) # LED ON else: GPIO.output(LED_PIN, False)# LED OFF time.sleep(0.1) except KeyboardInterrupt: GPIO.cleanup()
Always use a current-limiting resistor (typically 220-330Ω) when connecting an LED to a GPIO pin to prevent damage to both the LED and the Raspberry Pi.

SPI (Serial Peripheral Interface) on Raspberry Pi

Definition: SPI is a synchronous serial communication interface specification used for short-distance communication, primarily in embedded systems.

Characteristics:
  • Full Duplex: Can send and receive data simultaneously
  • Master-Slave Architecture: The Raspberry Pi acts as the Master, controlling the clock
  • High Speed: Generally faster than I2C
The 4 Wires:
  • MOSI (Master Out Slave In): Master (Pi) sends data to the Slave (Sensor)
  • MISO (Master In Slave Out): Slave (Sensor) sends data to the Master (Pi)
  • SCLK (Serial Clock): The Master generates the clock pulses to synchronize data transmission
  • CE/SS (Chip Enable / Slave Select): Used to select which slave device the master wants to communicate with. The Pi has two Chip Select pins (CE0, CE1) by default, allowing two SPI devices.
SPI must be enabled using the raspi-config utility before it can be used in Python programs.

Question 6. c) i) Explain about Python web application framework - Django. ii) Explain about Designing a RESTful web API.

Django Framework

Django is a high-level Python web framework that encourages rapid development and clean, pragmatic design.

Architecture:

It follows the MVT (Model-View-Template) pattern.

  • Model: Manages the data and database structure
  • View: Handles the business logic and interacts with the model
  • Template: Handles the presentation layer (HTML)
Use in IoT:

Django is excellent for building the central server that IoT devices report to. It can manage users, visualize data graphs, and store historical sensor logs in a robust database (like PostgreSQL).

Django's built-in admin interface makes it easy to create management panels for your IoT devices without writing additional code.

Designing a RESTful Web API

REST (Representational State Transfer) is an architectural style for designing networked applications. A RESTful API allows IoT devices to communicate with a server using standard HTTP methods.

Key Design Principles:
  • Resources: Everything is a resource (e.g., a sensor reading) identified by a URI (Uniform Resource Identifier)
    Example: http://myserver.com/api/sensors/temp1
  • Stateless: Each request from the client (IoT device) to server must contain all information needed to understand the request
HTTP Methods:
  • GET: Retrieve data (e.g., Get status of a light)
  • POST: Create data (e.g., Upload a new temperature reading)
  • PUT: Update data (e.g., Update the configuration of a device)
  • DELETE: Remove data
JSON Format:

Data is usually exchanged in JSON format because it is lightweight and easy for both Python and JavaScript to parse.

Question 1. c) Give overview of cloud storage models in brief?

Cloud Storage Models

Cloud storage models define how data is stored, accessed, and managed in the cloud environment. In the context of IoT, where devices generate massive amounts of unstructured and structured data, choosing the right model is critical.

Object Storage (e.g., Amazon S3):
  • Concept: Data is stored as distinct units called "objects" (files, images, logs) along with metadata and a unique identifier
  • Use in IoT: Ideal for storing vast amounts of unstructured data like sensor logs, video feeds from surveillance cameras, or firmware update files. It is highly scalable and accessible via HTTP APIs
Block Storage (e.g., Amazon EBS):
  • Concept: Storage is split into fixed-sized blocks. It acts like a physical hard drive attached to a virtual server
  • Use in IoT: Used by the virtual servers (like EC2 instances) running the IoT application backend or databases that require low-latency read/write operations
File Storage (e.g., Amazon EFS):
  • Concept: Data is stored in a hierarchical file and folder structure, similar to a standard PC file system (NFS/SMB)
  • Use in IoT: Useful when multiple IoT application servers need access to the same shared files simultaneously
Choosing the right storage model depends on your specific IoT use case, considering factors like access patterns, scalability requirements, and cost considerations.

Question 1. d) Explain any three basic APIs?

Three Basic Types of APIs

Local APIs (OS Level):
  • These are the services provided by the Operating System to applications running on the same device
  • Example: In an IoT device, the code uses local APIs to control the hardware (like GPIO pins on a Raspberry Pi) or access the file system
Web APIs (REST/SOAP):
  • These allow interaction between two different systems over a network (Internet), typically using HTTP
  • Example: An IoT device sending temperature data to a cloud server using a POST request, or a mobile app fetching that data using a GET request. This is the foundation of IoT cloud communication
Program APIs (Libraries/Packages):
  • These are sets of code libraries or packages that developers import to perform specific tasks without writing code from scratch
  • Example: Importing the json package in Python to parse data, or the boto3 library to interact with AWS
Understanding these different API types is crucial for IoT development, as devices often need to interact with local hardware, cloud services, and various software libraries.

Question 2. d) Explain the process of designing a RESTful web API?

Designing a RESTful Web API

Designing a RESTful API involves creating a standardized way for IoT devices and clients to communicate with the server.

Identify Resources:

Determine what "things" or data needs to be exposed. In IoT, resources could be Sensors, Devices, or Logs.

Define URIs (Uniform Resource Identifiers):

Assign unique endpoints to these resources.

Example: http://api.mysystem.com/sensors/temp_sensor_01

Choose HTTP Methods:

Map actions to standard HTTP verbs:

  • GET: Retrieve specific resource (e.g., Read temperature)
  • POST: Create a new resource (e.g., Upload new data)
  • PUT: Update an existing resource (e.g., Change device settings)
  • DELETE: Remove a resource
Determine Representation (Format):

Decide how data is exchanged. JSON is the standard for IoT because it is lightweight and human-readable.

Statelessness:

Ensure the server does not store the client's state between requests. Each request must contain all necessary information (authentication, data) to complete the task.

Well-designed RESTful APIs make it easier to integrate different IoT devices and applications, promoting interoperability and scalability.

Question 3. c) Illustrate the cloud for IoT?

Cloud for IoT Architecture

The "Cloud for IoT" refers to the ecosystem of services used to collect, store, and analyze data from physical devices.

Device Layer:

Sensors and actuators collect data from the physical environment.

Communication Layer:

Devices send data to the cloud via gateways using protocols like MQTT or HTTP.

Cloud Layer:
  • Ingestion: Accepts incoming data streams (e.g., AWS IoT Core)
  • Storage: Persists data (e.g., S3, DynamoDB)
  • Processing: Analyzes data (e.g., Lambda, EC2)
Application Layer:

Users visualize data via dashboards or mobile apps.

Key Benefit:

It offloads heavy computation and storage from the limited resource IoT devices to powerful cloud servers.

Cloud services enable IoT systems to scale globally, process massive amounts of data, and provide advanced analytics capabilities that would be impossible with on-premises solutions.

JSON Package in Python

Different Services of Cloud Computing

Cloud providers offer a vast array of services, categorized broadly into:

  • Storage Services: Object storage (S3), block storage, and databases to hold data
  • Networking Services: Virtual Private Clouds (VPC), Load Balancers to manage traffic
  • IoT Specific Services: Device management, message brokers (IoT Core), and analytics

Service Models of Cloud Computing

IaaS (Infrastructure as a Service):
  • Provides raw computing resources (VMs, storage, network)
  • User manages: OS, applications, data
  • Example: Amazon EC2
PaaS (Platform as a Service):
  • Provides a platform (runtime, tools) to develop and deploy applications without managing the underlying hardware or OS
  • User manages: Applications and data
  • Example: Google App Engine, AWS Elastic Beanstalk
SaaS (Software as a Service):
  • Delivers software applications over the internet
  • User manages: Nothing (just uses the software)
  • Example: Gmail, Dropbox, IoT Dashboards provided as a service

Question 5. c) Explain about weather reporting- air pollution monitoring

Weather Reporting & Air Pollution Monitoring

This is a classic IoT case study involving sensing, cloud transmission, and visualization.

System Architecture:
Sensing Node:
  • Sensors: Gas sensors (MQ-135 for Air Quality), PM2.5 sensors (dust), DHT11 (Temperature & Humidity)
  • Controller: Raspberry Pi or ESP8266 reads the sensor data
Data Processing:

The controller converts raw analog signals from sensors into digital values (e.g., PPM for gases).

Connectivity:

The device connects to Wi-Fi and sends the data to a Cloud platform (like ThingSpeak or AWS IoT) using REST API or MQTT.

Visualization:

The cloud platform plots the data on a real-time graph. Users can monitor pollution levels remotely via a web dashboard or mobile app.

Alerts:

If pollution exceeds a threshold, the system sends an email or SMS alert.

Such systems are crucial for urban planning and public health, providing real-time data that can help reduce pollution exposure and improve air quality.

Question 6. c) Explain about IoT in Agriculture

IoT in Agriculture (Smart Agriculture)

Smart Agriculture uses IoT to optimize farming practices, reduce waste, and increase yield.

Precision Farming:
  • Soil Monitoring: Moisture sensors (soil hygrometers) detect when the soil is dry
  • Automated Irrigation: The system automatically turns on water pumps only when needed, saving water
Crop Monitoring:
  • Drones or camera modules attached to Raspberry Pis monitor crop health using image processing (NDVI analysis) to detect diseases early
Greenhouse Automation:
  • Sensors control temperature, humidity, and light levels inside a greenhouse by automatically adjusting fans, heaters, and shades
Livestock Monitoring:
  • Wearable sensors track the health and location of cattle, alerting farmers if an animal is sick or straying
IoT in agriculture helps farmers make data-driven decisions, leading to increased productivity, reduced costs, and more sustainable farming practices.
